## Что сделать

Собери план простого NLP baseline для классификации текстов: например отзыв положительный/отрицательный, тикет в поддержку по категории или сообщение спам/не спам.

Тебе не нужно запускать код внутри приложения. Задача карточки — подготовить рабочий дизайн эксперимента:

- сформулируй, что является входом и что является целевой меткой;
- опиши минимальную очистку текста;
- выбери baseline-подход: `CountVectorizer` или `TfidfVectorizer` + простая модель;
- выбери метрику и объясни, почему она подходит;
- продумай 5 примеров ошибок, которые модель может делать.

## Как себя проверить

- Задача сформулирована так, что другой человек сможет собрать первый эксперимент.
- Метрика связана с реальной целью, а не выбрана случайно.
- Ты описал, как разделить данные на train/test без утечки.
- Есть план error analysis: какие тексты посмотреть после первого результата.

README эксперимента: постановка задачи, схема пайплайна, выбранная метрика, baseline, ожидаемые ошибки и план улучшений.
