# Coding Interview: Algorithm Mock

## Что сделать

Проведи короткий coding interview mock на одну algorithm-задачу. Выбери естественный паттерн: hashmap, two pointers, sliding window, stack или recursion. Не начинай сразу с кода: сначала запиши ход мысли.

Шаблон:

```text
Problem:
Input:
Output:
Brute force idea:
Brute force complexity:
Optimized pattern:
Optimized idea:
Edge cases:
Final Big O:
Communication self-review:
```

После этого реши задачу в Notebook, Tasks или отдельной ячейке. Минимум один раз проговори решение вслух так, как на интервью: что уточняешь, почему выбран паттерн, что проверяешь тестами.

## Как себя проверить

- Ты уточнил input/output до кода.
- Brute force idea записана честно, даже если она медленная.
- Optimized idea объясняет, какая структура данных убирает лишнюю работу.
- Есть минимум 3 edge cases.
- Есть Big O для времени и памяти.
- Ты можешь объяснить решение без чтения кода строка за строкой.
- В self-review есть один пункт `retry later`, если решение заняло слишком много времени или объяснение было слабым.

## Что положить в портфолио

Мини-лог `coding_interview_log.md`: задача, выбранный pattern, brute force, optimized solution, asserts, complexity, edge cases и короткий self-review. Не добавляй чужие premium-условия задач; формулируй свою учебную версию.
