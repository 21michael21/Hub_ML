# LLM Prompt Review Drill

## Что сделать

Возьми слабый prompt и перепиши его так, чтобы LLM получил ясную задачу, контекст, ограничения, формат ответа и критерии качества.

Слабый prompt:

```text
Посмотри мой анализ заказов и скажи, что улучшить.
```

Сделай новую версию prompt для ситуации: у тебя есть EDA по `df_orders.csv`, график распределения `order_sum` и таблица пользователей с количеством заказов. Prompt должен просить LLM не придумывать метрики, отделять факты от гипотез и дать короткий список следующих проверок.

Прогони оба prompt во внешнем бесплатном чате вручную и сравни ответы.

## Как себя проверить

- В prompt явно есть задача и роль LLM.
- Есть контекст: какие данные/артефакты пользователь показывает.
- Есть ограничения: не invent metrics, не делать выводы без evidence.
- Есть формат ответа: например observations, hypotheses, next checks, caveats.
- После сравнения ты можешь объяснить, почему второй ответ полезнее первого.

## Что положить в портфолио

Мини-артефакт "Before/After Prompt Review": слабый prompt, улучшенный prompt, два ответа модели, твой разбор качества и список правил, которые ты теперь используешь для рабочих LLM prompts.
