# Resume Remote Career: Honest Hub_ML Bullets

## Что сделать

Составь 5 честных resume bullets из реальной работы в Hub_ML. Не придумывай опыт, seniority, salary claims, production impact или метрики, которых нет в проектах.

Для каждого bullet заполни evidence map:

```text
Bullet:
Project:
Technical method:
Evidence file:
Commit or test:
Artifact:
Weakness:
Rewrite:
```

Подсказка: сильный bullet обычно содержит action verb, technical method и artifact/result. Например, если ты сделал Data Lab project, bullet может ссылаться на README, chart, report, test или model card. Если evidence нет, пометь bullet как слабый и перепиши его скромнее.

## Как себя проверить

- Есть 5 bullets, и каждый связан с реальным Hub_ML artifact.
- В каждом bullet есть action verb.
- В каждом bullet есть technical method: pandas, SQL, matplotlib, baseline, tests, README, experiment log и т.п.
- В каждом bullet есть artifact/result, который можно открыть или создать.
- Минимум один слабый bullet переписан в честную версию.
- Нет фейкового business impact, salary claims, seniority claims или production-ready заявлений.
- Есть план, какой README/chart/model card/experiment log нужно доделать для усиления резюме.

## Что положить в портфолио

Файл `resume_remote_bullets.md`: 5 bullets, evidence map, weak bullet before/after и список missing evidence. Этот файл можно использовать как черновик для резюме и remote interview stories.
