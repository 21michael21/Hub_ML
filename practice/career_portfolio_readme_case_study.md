# Portfolio README Case Study

## Что сделать

Выбери один завершённый Hub_ML project или practice card и преврати его в portfolio README outline. Это не генерация фейковых результатов: если метрика, график или вывод ещё не проверены, оставь честную пометку `to be filled after analysis`.

Заполни структуру:

```markdown
# Project title

## Problem

## Data

## Approach

## Results

## Limitations

## Reproducibility

## Artifacts

## Next Steps

## Resume Bullet Draft
```

Добавь project summary в 3-5 предложениях, список artifacts to include и один resume bullet draft. В bullet не придумывай impact: пиши только то, что подтверждено проектом, тестами, отчётом, chart или experiment run.

## Как себя проверить

- README можно понять за 30 секунд.
- Есть problem, data, approach, results, limitations, reproducibility и next steps.
- В results нет fake metrics или неподтверждённых claims.
- В limitations есть минимум 3 честных ограничения.
- Artifacts перечислены явно: notebook, report, chart, table, model card или experiment log.
- Reproducibility использует относительные пути и не требует приватных данных.
- Resume bullet имеет структуру action + method + artifact/result.

## Что положить в портфолио

Файл `portfolio_case_study.md` или `README.md`, который можно позже перенести в GitHub repository. Добавь ссылки на реальные artifacts только после того, как они созданы и проверены.
