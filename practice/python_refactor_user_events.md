## Что сделать

Представь, что тебе дали список событий пользователя:

```python
events = [
    {"user_id": 1, "event": "open_app", "source": "organic"},
    {"user_id": 1, "event": "purchase", "source": "organic"},
    {"user_id": 2, "event": "open_app", "source": "ads"},
]
```

Напиши функцию `summarize_events(events)`, которая возвращает словарь:

- сколько всего событий было у каждого пользователя;
- какие типы событий встретились;
- какие источники трафика встретились.

После первой рабочей версии отрефактори код: убери дублирование, дай понятные имена переменным, раздели маленькие шаги на вспомогательные функции, если это делает код читабельнее.

## Как себя проверить

- Функция понятно читается через неделю без комментариев на каждой строке.
- Пустой список событий не ломает код.
- Если у события нет `source`, ты явно решил, что делать: пропустить, записать `"unknown"` или поднять ошибку.
- Результат можно объяснить человеку, который не видел исходный код.
